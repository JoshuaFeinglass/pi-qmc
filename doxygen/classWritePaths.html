Detailed Description

Algorithm for writting paths to an ascii file.

Last line of the file is the same positions as the first lines, just with the net permuation.

If there are clones (see MPIManager::nclone), then each clone gets its own file, with its cloneID in the name.

To handle mulitiple workers (see MPIManager::nworker) writing is done in stages. Only the first worker writes. The first worker writes from its lowest sample slice to half its highest stored slice. Then the beads are shifted. This process continues until all beads are written.

Because of the uni-directionality of shift, beads are written in reverse time order.
